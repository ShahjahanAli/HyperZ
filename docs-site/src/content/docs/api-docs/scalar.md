## Overview

HyperZ includes **Scalar** as a modern alternative to Swagger UI for browsing your OpenAPI documentation. Scalar provides a beautiful, interactive API reference with built-in code samples, request testing, and dark mode.

## Access

When the dev server is running:

- **Scalar Reference:** `http://localhost:7700/api/_admin/reference`
- **Swagger UI:** `http://localhost:7700/api/_admin/docs`
- **OpenAPI JSON:** `http://localhost:7700/api/_admin/reference/openapi.json`

## Configuration

Configure Scalar in `config/docs.ts`:

```typescript
export default {
    // Swagger UI settings
    enabled: true,
    path: '/api/docs',
    title: 'My API Documentation',
    // ...

    // Scalar settings
    scalar: {
        enabled: true,
        path: '/api/reference',
        theme: 'purple',         // 'default' | 'alternate' | 'moon' | 'purple' | 'solarized' | etc.
        showSidebar: true,
        hideDownloadButton: false,
        hideDarkModeToggle: false,
        defaultHttpClient: {
            targetKey: 'javascript',
            clientKey: 'fetch',
        },
    },
};
```

## Environment Variables

```bash
DOCS_ENABLED=true          # Enable/disable all documentation
SCALAR_ENABLED=true        # Enable/disable Scalar specifically
SCALAR_THEME=purple        # Scalar theme
```

## Available Themes

| Theme | Description |
|-------|-------------|
| `default` | Scalar's default light/dark theme |
| `alternate` | Alternative clean design |
| `moon` | Dark theme with blue accents |
| `purple` | Purple-accented theme (HyperZ default) |
| `solarized` | Solarized color palette |
| `bluePlanet` | Blue space-inspired theme |
| `saturn` | Saturn-inspired dark theme |
| `kepler` | Astronomical theme |
| `mars` | Red/Mars-inspired theme |
| `deepSpace` | Deep dark space theme |
| `none` | No built-in theme (fully custom) |

## Auto-Generated Documentation

Scalar reads the same OpenAPI 3.1 spec generated by HyperZ, which includes:

- **All registered routes** — automatically discovered from your route files
- **Request/response schemas** — derived from Zod validation schemas
- **Authentication** — JWT Bearer and API Key schemes
- **Tags** — auto-grouped by URL path segment

### Decorators for Extra Documentation

Use decorators on controller methods for richer docs:

```typescript
import { Summary, Tags, Response } from '../../src/docs/Decorators.js';

class ProductController extends Controller {
    @Summary('List all products')
    @Tags('Products')
    @Response(200, 'Array of products')
    async index(req: Request, res: Response) {
        // ...
    }
}
```

## Custom CSS

Inject custom styles via `config/docs.ts`:

```typescript
scalar: {
    customCss: `
        .scalar-app { font-family: 'Inter', sans-serif; }
        .sidebar { background: #0a0a0f; }
    `,
}
```
